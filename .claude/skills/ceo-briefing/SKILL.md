# Skill: CEO Briefing

Generate a weekly CEO Briefing report combining data from Odoo accounting, Facebook insights, vault activity, and task completion metrics.

## When to Use
Invoke with `/ceo-briefing` to generate a comprehensive weekly business briefing. Can also be triggered by a scheduled cron job every Sunday night.

## Instructions

### Step 1: Gather Financial Data from Odoo
Use the `get_ceo_briefing_data` MCP tool from the Odoo MCP server to fetch:
- Revenue for the period (default: last 7 days)
- Outstanding invoices
- Overdue payments
- New contacts/customers

If Odoo is not available (Docker not running), note this in the briefing and skip financial data.

### Step 2: Gather Social Media Metrics
Use the `get_page_insights` and `list_page_posts` MCP tools from the Facebook MCP server to fetch:
- Page engagement metrics (views, reach, engagement)
- Recent post performance (likes, comments, shares)
- Follower growth

If Facebook is not configured, note this in the briefing and skip social metrics.

### Step 3: Gather Vault Activity
Read the log files from the past 7 days in `AI_Employee_Vault/Logs/`:
- Count total items processed
- Count emails handled
- Count approvals pending
- Count tasks completed (items moved to Done/)
- Identify any stale items (in Needs_Action/ for >24 hours)

### Step 4: Read Business Goals
Read `AI_Employee_Vault/Business_Goals.md` (if it exists) to compare actual performance against targets.

### Step 5: Generate the Briefing
Create a file in `AI_Employee_Vault/Briefings/` named `<YYYY-MM-DD>_CEO_Briefing.md`.

If the `Briefings/` folder does not exist, create it.

Use this format:

```markdown
---
generated: "<ISO 8601 timestamp>"
period: "<start date> to <end date>"
type: ceo_briefing
---

# CEO Briefing - Week of <date>

## Executive Summary
<2-3 sentence high-level overview of the week>

## Financial Overview
- **Revenue This Week:** $<amount>
- **Invoices Issued:** <count>
- **Outstanding Receivables:** $<amount>
- **Overdue Payments:** <count> totaling $<amount>
- **Collection Rate:** <percentage>

## Social Media Performance
### Facebook
- **Page Views:** <count>
- **Post Reach:** <count>
- **Engagement:** <count> interactions
- **Top Post:** "<post summary>" (<likes> likes, <comments> comments)
- **Follower Growth:** +<count>

## AI Employee Activity
- **Items Processed:** <count>
- **Emails Handled:** <count>
- **Tasks Completed:** <count>
- **Approvals Pending:** <count>
- **Avg Processing Time:** <estimate>

## Alerts & Action Items
- [ ] <any overdue invoices to follow up>
- [ ] <any stale items needing attention>
- [ ] <any system issues observed>

## Proactive Suggestions
- <cost optimization opportunities>
- <upcoming deadlines>
- <growth opportunities>

---
*Generated by AI Employee - Powered by Claude Code*
*Briefing period: <start> to <end>*
```

### Step 6: Log the Action
Append to today's log file:
```
## HH:MM:SS - CEO Briefing Generated
- **Type:** ceo_briefing
- **Period:** <start> to <end>
- **Result:** success
```

### Step 7: Update Dashboard
Update `AI_Employee_Vault/Dashboard.md` to note the latest briefing was generated.

### Step 8: Inform the User
Tell the user:
1. The briefing has been saved to `Briefings/`
2. Highlight any critical alerts (overdue payments, stale items)
3. Summarize the key numbers (revenue, items processed)
